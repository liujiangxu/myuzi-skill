# 每日推荐任务

用户输入: $ARGUMENTS

## 自然语言触发

当用户说以下任一意图时，执行本任务：

- `每日推荐`
- `今日推荐`
- `每天推荐 10 只股票`
- `推荐 10 只优选股票`
- `今天有什么股票值得看`
- `给我每日优选股票`

## 工作流

优先执行脚本：

```bash
cd skills/deep-analysis/scripts
python daily_recommend.py
```

可选参数：

```bash
python daily_recommend.py --top 10
python daily_recommend.py --top 10 --json
```

脚本会：

1. 拉取 A 股全市场快照。
2. 过滤 ST、退市、成交过低、极端涨跌停、微盘流动性不足标的。
3. 按估值、动量、流动性、市值弹性、风险惩罚做综合评分。
4. 输出 Top 10 优选股票。
5. 写入 `.cache/_global/daily_recommendations.json` 供后续审计和复用。

## 输出要求

直接向用户输出 Markdown 表格，包含：

| 字段 | 要求 |
|---|---|
| 排名 | 1-10 |
| 股票 | 名称 + 标准代码 |
| 风格 | 价值低估 / 动量增强 / 白马稳健 / 均衡配置 |
| 综合分 | 0-100 |
| 价格 | 最新价 |
| 涨跌幅 | 当日涨跌幅 |
| PE / PB | 估值指标 |
| 推荐理由 | 1-3 条简单但具体的理由 |
| 主要风险 | 至少 1 条 |

## 重要说明

- 每日推荐是“全市场预筛 + 快速完全评估候选”，不是最终投资建议。
- 输出 10 只后，必须提示用户：如要正式买入，应对感兴趣标的继续说 `完整深度分析 <代码>`，再跑 22 维数据、51 评委、17 种机构方法和 HTML 报告。
- 不要凭空编造财务数据；推荐理由只能使用脚本输出里的估值、涨跌幅、市值、换手率、成交额等字段。

## 兜底

如果 akshare 不可用或网络失败：

1. 明确告诉用户当前环境无法拉取全市场快照。
2. 建议先安装依赖或稍后重试：`pip install -r requirements.txt`。
3. 不要编造 10 只股票清单。
